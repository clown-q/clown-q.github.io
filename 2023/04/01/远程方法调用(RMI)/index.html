# 前言

RMI (Remote Method Invocation) 模型是一种分布式对象应用，使用 RMI 技术可以使一个 JVM 中的对象，调用另一个 JVM 中的对象方法并获取调用结果。这里的另一个 JVM 可以在同一台计算机也可以是远程计算机。因此，RMI 意味着需要一个 Server 端和一个 Client 端。实际上就是在一个 java 虚拟机上调用另一个 java 虚拟机的对象上的方法 RMI传输数据序列化后的数据

# RMI 流程

这里使用 p 牛的文档中的代码来学习 RMI 的流程

Server 端代码

定义了一个远程接口 IRemoteHelloWorld ，其中包含一个 hello() 方法，用于向客户端发送问候语。然后，通过实现这个接口的 RemoteHelloWorld 类来提供实现。在 RemoteHelloWorld 类中，重写了 hello() 方法，当客户端调用这个方法时，会输出 "访问成功" 并返回 "Hello world"。在 RMIServer 类中，通过 start() 方法来启动 RMI 服务。在这个方法中，首先创建了一个 RemoteHelloWorld 对象，然后创建了一个 RMI 注册表，并将 RemoteHelloWorld 对象注册到这个注册表中，以便客户端可以通过注册表来查找和调用这个对象的方法。最后，在 main() 方法中，创建了一个 RMIServer 对象并调用了 start() 方法来启动 RMI 服务。这里 Naming.rebind ("rmi://127.0.0.1:1099/Hello", h); 里的 rmi 可以不写，端口默认就死 1099，如果不使用默认的就必须写

这里记录一下上面有关 extends 与 implements 的区别

  1. 在类的声明中，通过关键字 extends 来创建一个类的子类。一个类通过关键字 implements 声明自己使用一个或者多个接口
  2. extends 是继承某个类，继承之后可以使用父类的方法，也可以重写父类的方法；implements 是实现多个接口，接口的方法一般为空的，必须重写才能使用
  3. extends 是继承父类，只要那个类不是声明为 final 或者那个类定义为 abstract 的就能继承，JAVA 中不支持多重继承，但是可以用接口 来实现，这样就要用到 implements，继承只能继承一个类，但 implements 可以实现多个接口，用逗号分开就行了

Client 端：

通过 RMI 协议查找名为 "Hello" 的远程对象，并将其转换为 IRemoteHelloWorld 类型的 hello 对象。然后调用 hello 对象的 hello 方法，将返回值赋给 ret 变量，并将其输出。这里也是 rmi：可以不写，端口也是使用默认的可以不写

# RMI 利用 codebase 代码执行

codebase 是一个地址，一个高数 JVM 去哪个地方搜索类，与 CLASSPATH 不同的是，CLASSPATH 是本地路径，而 codebase 是远程的 URL，如果我们指定 codebase= http://example.com/ ，然后加载 org.vulhub.example.Example 类，则 Java 虚拟机会下载这个文件 http://example.com/org/vulhub/example/Example.class ，并作为 Example 类的字节码。

在 RMI 中可以将 codebase 随着序列化一同传输，服务器在接受到数据后反序列化的过程中发现对象就会去 CLASSPATH 中寻找类，如果没有就会去 codebase 远程加载类，这里控制这个 codebase 就可以执行任意代码
